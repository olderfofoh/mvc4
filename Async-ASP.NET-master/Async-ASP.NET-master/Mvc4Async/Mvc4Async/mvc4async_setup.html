1. Run the Web API server application (WebAPIpgw). The project is set to use IIS Express with port 7734 (http://localhost:7734/). You can run the WebAPIpgw on local host or on a remote server. The http://localhost:7734/Notes.html will come up with the following test URLs.
    • http://localhost:7734/api/gizmos
    • http://localhost:7734/api/products
    • http://localhost:7734/api/widgets
    • http://localhost/WebAPIpgw/api/gizmos
    • http://localhost/WebAPIpgw/api/products
    • http://localhost/WebAPIpgw/api/widgets
  2. If you change the WebAPIpgw, record it the root URI for the MVC4Async application. For example if hosted on a computer named kaVS430, the root URI would be http://kaVS430/WebAPIpgw/ and you would store that value in the MVC4Async application Web.config file as
     

    <appSettings>

        <add key="WidgetServiceURI" value="http://kaVS430/WebAPIpgw/" />

        <!--   items removed à
     </appSettings>

  3. Run the MVC4Async application.
  4. To control the simulated latency, change the delay in the GetDelay method in the  WebAPIpgw\Utility\Util.cs file.
